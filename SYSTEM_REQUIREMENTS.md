# מסמך אפיון מערכת (PRD) - AS-CRM

## סקירה כללית
מערכת ה-CRM היא פלטפורמת תוכנה כשירות (SaaS) מרובת משתמשים (Multi-tenant), המיועדת לעסקים לניהול קשרי לקוחות ותהליכי מכירה. המערכת מספקת לוח בקרה מרכזי בשפה העברית הכולל ניהול לידים בתצוגת פייפליין, מעקב אחר משימות והתקשרויות, טפסים דיגיטליים לקליטת לידים, וניהול הגדרות מותאמות אישית לכל עסק.

## מודל נתונים (Data Models)
המערכת מבוססת על בסיס הנתונים Supabase וכוללת את הישויות המרכזיות הבאות:

1. **פרופילים (Profiles / Businesses):** מייצג עסק או משתמש רשום במערכת.
   - במשתמש: `id` (UUID מזהה משתמש), `business_name` (שם העסק), `slug` (מזהה טופס פומבי), `is_approved` (האם החשבון מאושר על ידי ההנהלה), `settings` (אובייקט JSON המכיל צבעי מותג, סטאטוסים מותאמים אישית, הגדרות התראות, מכסת AI, וסטטוס תשלום).

2. **לקוחות / לידים (Clients):** מייצג איש קשר או ליד המשויך לעסק מסוים.
   - שדות: `id`, `user_id` (שיוך לעסק), `status` (סטטוס בפייפליין, למשל: "חדש", "בטיפול", "סגור"), `contact` (שם הלקוח), `phone` (טלפון), `email` (דוא"ל), `company` (שם חברה), `role` (תפקיד), `history` (היסטוריית טיפול כללית), `nextCall` (תאריך ושעת תזכורת), `is_vip` (האם הלקוח מוגדר כ-VIP).

3. **קבצי לקוחות (Client Files):** מייצג מסמכים מצורפים לכרטיס לקוח.
   - שדות: `id`, `client_id` (שיוך לליד), `file_name` (שם הקובץ), `file_url` (כתובת הקובץ ב-Storage), `created_at`.

4. **תיעוד שיחות / ציר זמן (Client Logs):** היסטוריית התקשרויות מפורטת לכל לקוח.
   - שדות: `id`, `client_id` (שיוך לליד), `note_content` (תוכן ההערה), `created_at`.

## עמודים ורכיבי ממשק
1. **דשבורד ראשי (`Dashboard.jsx`):** מסך העבודה המרכזי. כולל אריחי סטטיסטיקה, שורת חיפוש ופעולות (ייצוא/ייבוא אקסל, הוספת לקוח), וטבלאות פייפליין המחולקות לפי סטטוסים ומשימות דחופות להיום.
2. **פרופיל לקוח (`ClientProfile.jsx`):** חלונית צד (Drawer) להצגת ועריכת פרטי ליד ספציפי. כוללת אפשרות לעדכון פרטים, הוספת הערות, קביעת תזכורות, העלאת קבצים, ייצוא ל-PDF ושליחת הודעות וואטסאפ.
3. **טופס לידים פומבי (`PublicLeadForm.jsx`):** דף נחיתה מותאם אישית לכל עסק (מבוסס slug) המאפשר ללקוחות פוטנציאליים להשאיר פרטים שיוזנו אוטומטית כליד "חדש" במערכת של אותו עסק.
4. **דשבורד ניהול מערכת (`AdminDashboard.jsx`):** מסך ייעודי ל-Master Admin (מנהל על). מאפשר צפייה בכל בתי העסק הרשומים, אישור/חסימת משתמשים, מעקב אחר תשלומי הקמה או מנוי חודשי, ושינוי הרשאות/סטטוס מנוי ללקוחות תוכנה.
5. **מסכי התחברות והרשמה (`LoginPage.jsx`):** כניסה והרשמה למערכת באמצעות אימייל.
6. **אשפי הגדרה (`SetupWizard.jsx`, `OnboardingWizard.jsx`):** תהליכי קליטה המנחים משתמש חדש בהגדרת העסק, בחירת שם, צבע מותג וכתובת URL לטופס הלידים.
7. **הגדרות (`Settings.jsx`):** ניהול צבעי המותג, התראות קופצות, עריכת תיאור העסק, ועריכת סטטוסים מותאמים אישית לפייפליין.

## תהליכי ליבה (Core User Flows)
1. **קליטת עסק חדש למערכת (Onboarding):** משתמש נרשם למערכת, עובר מסכי הגדרה ראשונית לאפיון העסק, וממתין לאישור מנהל (Master Admin) אשר מפעיל לו את החשבון לאחר בחינת התשלום.
2. **לכידת ליד חיצוני:** לקוח קצה מבקר ב"טופס לידים פומבי" (URL ייעודי של העסק), ממלא פרטים, והליד נוצר אוטומטית במסד הנתונים תחת סטטוס "חדש" בלוח הבקרה של בעל העסק.
3. **ניהול וקידום עסקאות:** בעל העסק מתחבר לדשבורד, מבחין בליד חדש או תזכורת להיום, פותח את כרטיס הלקוח, מתעד שיחה (Client Logs), קובע תזכורת להמשך קשר (`nextCall`), ומעדכן את סטטוס הליד (למשל מ"חדש" ל"בטיפול" או "סגור").
4. **ייצור דוחות וגיבוי:** המשתמש מפעיל ייצוא של נתוני הלקוחות לקובץ Excel מתוך הדשבורד, או מייבא רשימת לקוחות קיימת דרך מודל ייבוא ה-CSV.

## זיהוי צווארי בקבוק (Bottlenecks)
בסריקת הקוד אותרו מספר אזורים מרכזיים שמכילים לוגיקה מורכבת וסבוכה שככל הנראה מובילה לבאגים ולקשיי תחזוקה:
1. **`App.jsx`:** הקובץ מועמס באחריות רבה - ניתוב (Routing), ניהול State גלובלי (לקוחות, פרטי משתמש, סשן), משיכת נתונים ושליחת התראות דחיפה (Push notifications). היעדר הפרדה (כגון שימוש ב-Context API או Custom Hooks) גורם לרינדורים מיותרים וקוד ספגטי.
2. **`Dashboard.jsx`:** מכיל ערבוב של תצוגת UI, ניתוח נתונים (חישובי פייפליין, מיונים), עדכונים אופטימיסטיים ישירים, ושמירת נתונים ל-Supabase. העדכונים האופטימיסטיים ללא סינכרון עמוק עלולים לגרום לנתונים להיות לא מעודכנים מול השרת.
3. **`AdminDashboard.jsx`:** למרות שהוא מיועד ככלי פנימי, הלוגיקה שלו משלבת הפעלת פונקציות קצה (Edge Functions), עדכוני ממשק ידניים ומורכבים (Optimistic updates for Toggles), וסטייל תצוגתי כבד בתוך קובץ אחד ללא רכיבי-משנה מופרדים.
